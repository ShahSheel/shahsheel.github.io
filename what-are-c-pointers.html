Introduction

If you are learning C and do not understand pointers, then is a great article for you. In this article, I shall be explaining the bullets below in a easy to understand way with examples:

  1. Why do we use this?
  2. What we use it for
  3. How do I implement this?

The why....

Back in the days, in the year 1972 C was created (As well as the first ever scientifc calculator!). In those days we did not have power-crunching-computers that we have now and memory and speed was the upmost importance. Therefore pointers were introduced to help with dynamic memory allocation and the speed!

So what do we use it for?

An example of an application we use pointers in are microcontrollers. Otherwise, as a program we use it for:

  • To create dynamic data structures.
  • To pass and handle variable parameters passed to functions.
  • To access information stored in arrays. (Especially if you work with links).

Great! So how do i implement this?

To create dynamic data structures(Array example)

Unlike Java we do not have the beauty of having arraylists that supports dynamic memory allocation. This is where pointers come in for C - how do we do this?

Instead of claiming the memory up-front by doing array[50], the memory is allocated (from the heap) while the program is running. So the exact amount of memory is claimed and there is no waste. Even better, memory not used will be returned to the heap. (Freed memory can be used for other programs).


We don't use pointers for arrays only, in fact we use it for nearly every data-structure such as binary trees - which also is a great data structure to learn pointers!.

To pass and handle variable parameters passed to functions.

If you have seen other tutorials and pages, this is generally referred to Pointers as variables. But they only show examples of the program as a main(Which to me does not help!).

So basically, we use pointers as parameters for the variables. (Passing the address value to a pointer parameter which then stores a new value).

To access information stored in arrays.

Instead of doing the popular convention a[3] we can instead access the array by using a pointer. You may be thinking, why do we do that? Well - lets take a look back at why we use pointers.

Pointers are used to for dynamic allocation of memory and speed

If you havn't guessed yet, we are now moving onto the "speed part". When we access the elements via a pointer we are essentially copying it at run time. During the run time, we can perform any operation we wish to, such as sorting the array by lowest to highest (By accessing it via pointers) which then makes the array sorted.

It is faster to point to the array rather than having the array to access memory every time and then sort it.
